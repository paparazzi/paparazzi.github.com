imu_navgo module

NAVGO IMU.

  • Accelerometer: ADXL345 via I2C
  • Gyroscope: ITG3200 via I2C
  • Magnetometer: HMC58xx via I2C

Example for airframe file

Add to your firmware section:

<module name="imu_navgo"/>

Auto-loaded modules

The following modules are automatically loaded (just as if you had added them in the airframe file)

  • imu_common module
  • imu_nps module

Module functions

Init Functions

These initialization functions are called once on startup.

  • imu_navgo_init()

Event Functions

These event functions are called in each cycle of the module event loop.

  • imu_navgo_event()

Periodic Functions

These functions are called periodically at the specified frequency from the module periodic loop.

  • imu_navgo_periodic()
    • Running at maximum module frequency.
    • Autorun: LOCK
      Periodic function automatically starts after init and can't be stopped.

Raw imu_navgo.xml file:

<!DOCTYPE module SYSTEM "module.dtd">
<module name="imu_navgo" dir="imu">
<doc>
<description>
NAVGO IMU.
- Accelerometer: ADXL345 via I2C
- Gyroscope: ITG3200 via I2C
- Magnetometer: HMC58xx via I2C
</description>
</doc>
<autoload name="imu_common"/>
<autoload name="imu_nps"/>
<header>
<file name="imu_navgo.h" dir="boards/navgo"/>
</header>
<init fun="imu_navgo_init()"/>
<periodic fun="imu_navgo_periodic()"/>
<event fun="imu_navgo_event()"/>
<makefile target="!sim|nps|fbw">
<define name="IMU_NAVGO_I2C_DEV" value="i2c1"/>
<define name="USE_I2C1"/>
<define name="I2C1_SCLL" value="25"/>
<define name="I2C1_SCLH" value="25"/>
<define name="IMU_TYPE_H" value="boards/navgo/imu_navgo.h" type="string"/>
<file name="adxl345_i2c.c" dir="peripherals"/>
<file name="itg3200.c" dir="peripherals"/>
<file name="hmc58xx.c" dir="peripherals"/>
<file name="imu_navgo.c" dir="boards/navgo"/>
</makefile>
</module>
