tag_tracking module

Track position of a tag (ArUco, QRcode, ...) detected by an onboard camera

The tag detection and pose computation is done outside of the module, only the estimation by fusion of AHRS and visual detection with a Kalman filter is performed in this module

Example for airframe file

Add to your firmware section:

<module name="tag_tracking"/>

Module configuration options

Airframe file section

  • section name: TAG_TRACKING prefix: TAG_TRACKING_
    • name WPS value: [{.wp_id=WP_TAG, .tag_id=42}, ...]
      Description: Array of structs. If defined, update the position of the waypoints at the location of the tags
    • name SIM_WP value: WP_TARGET
      Description: designate a waypoint to be used as virtual tag in simulation; if TARGET waypoint exists in flight plan, it is used by default
    • name BODY_TO_CAM_PHI value: 0.
      Description: rotation from body to cam frame (phi)
    • name BODY_TO_CAM_THETA value: 0.
      Description: rotation from body to cam frame (theta)
    • name BODY_TO_CAM_PSI value: 0.
      Description: rotation from body to cam frame (psi)
    • name CAM_POS_X value: 0.
      Description: camera position in meter in body frame (X axis)
    • name CAM_POS_Y value: 0.
      Description: camera position in meter in body frame (Y axis)
    • name CAM_POS_Z value: 0.
      Description: camera position in meter in body frame (Z axis)
    • name COORD_TO_M value: 1e-3
      Description: transformation from camera coordinates to meters (default: mm to m)
    • name R value: 1
      Description: process noise in kalman filter
    • name Q_SIGMA2 value: 1.
      Description: measurement noise in kalman filter
    • name P0_POS value: 10.
      Description: initial covariance on position
    • name P0_SPEED value: 10.
      Description: initial covariance on speed
    • name TIMEOUT value: 5.
      Description: timeout for lost tracking in seconds
    • name PREDICT_TIME value: 1.
      Description: position prediction time when moving waypoint
    • name MAX_OFFSET value: 2.
      Description: maximum position offset for predicted positin compared to estimated position
    • name MAX_SPEED value: 4.
      Description: maximum horiontal speed for the tracking control
    • name MAX_VZ value: 2.
      Description: maximum vertical speed for the tracking control
    • name KP value: 0.5
      Description: horizontal gain on position error
    • name KPZ value: 0.2
      Description: vertical gain on position error

Module functions

Init Functions

These initialization functions are called once on startup.

  • tag_tracking_init()

Periodic Functions

These functions are called periodically at the specified frequency from the module periodic loop.

  • tag_tracking_propagate()
    • Frequency in Hz: 10.0
    • Autorun: TRUE
      Periodic function automatically starts after init.
    • Start function: tag_tracking_propagate_start()
      Executed before the periodic function starts.
  • tag_tracking_report()
    • Frequency in Hz: 1.0
    • Autorun: TRUE
      Periodic function automatically starts after init.

Datalink Functions

Whenever the specified datalink message is received, the corresponing handler function is called.

  • on message TARGET_POS call tag_tracking_parse_target_pos(buf)
