uwb_positioning module

Estimate local position from UWB anchor measurements.

UWB devices can be used for communication, ranging, and positioning. This module consumes anchor measurements and estimates a local position. With range measurements, 3 anchors are required/supported by the current trilateration path.

See https://hal-enac.archives-ouvertes.fr/hal-01936955 for more information on the EKF filtering.

Example for airframe file

Add to your firmware section: This example contains all possible configuration options, not all of them are mandatory!

<module name="uwb_positioning">
b'<configure name="UWB_POSITIONING_USE_AS_LOCAL_POS" value="FALSE|TRUE" />\n '
b'<configure name="UWB_POSITIONING_USE_AS_GPS" value="FALSE|TRUE" />\n '
</module>

Module configuration options

Configure Options

  • name: UWB_POSITIONING_USE_AS_LOCAL_POS value: FALSE|TRUE
    Description: use as a local positioning system (default: TRUE)
  • name: UWB_POSITIONING_USE_AS_GPS value: FALSE|TRUE
    Description: use as a fake GPS positioning system (default: FALSE)

Airframe file section

  • section name: UWB_POSITIONING prefix: UWB_POSITIONING_
    • name ANCHORS_IDS value: 1, 2, 3
      Description: Comma separated list of anchors ID
    • name ANCHORS_POS_X value: 0., 0., 5.
      Description: Comma separated list of anchors ID over X axis
    • name ANCHORS_POS_Y value: 0., 5., 0.
      Description: Comma separated list of anchors ID over Y axis
    • name ANCHORS_POS_Z value: 0., 0., 0.
      Description: Comma separated list of anchors ID over Z axis
    • name RANGE_OFFSET value: 0., 0., 0.
      Description: Offset applied to individual range measurements
    • name RANGE_SCALE value: 1., 1., 1.
      Description: Scale factor applied to individual range measurements
    • name INITIAL_HEADING value: 0.
      Description: Initial heading correction between anchors frame and global frame
    • name NB_ANCHORS value: 3
      Description: Set number of anchors, only 3 are required/supported at the moment
    • name USE_EKF value: FALSE|TRUE
      Description: Enable EKF filtering, required to estimate speed
    • name EKF_Q value: 1.0
      Description: EKF process noise
    • name EKF_R_DIST value: 0.1
      Description: EKF noise on distance measurements
    • name EKF_R_SPEED value: 0.1
      Description: EKF noise on speed measurements (if available)
    • name NOISE_X|Y|Z value: 0.1
      Description: Noise level reported by the POSITION_ESTIMATE message when USE_AS_LOCAL_POS is activated
    • name VEL_NOISE_X|Y|Z value: 0.1
      Description: Noise level reported by the VELOCITY_ESTIMATE message when USE_AS_LOCAL_POS is activated

Module functions

Init Functions

These initialization functions are called once on startup.

  • uwb_positioning_init()

Periodic Functions

These functions are called periodically at the specified frequency from the module periodic loop.

  • uwb_positioning_periodic()
    • Frequency in Hz: 10
    • Autorun: LOCK
      Periodic function automatically starts after init and can't be stopped.
  • uwb_positioning_range_periodic()
    • Frequency in Hz: 10
    • Autorun: LOCK
      Periodic function automatically starts after init and can't be stopped.
  • uwb_positioning_report()
    • Frequency in Hz: 10
    • Autorun: FALSE
      Periodic function is started by user command.
  • uwb_positioning_reset_heading_ref()
    • Frequency in Hz: 1
    • Autorun: FALSE
      Periodic function is started by user command.
